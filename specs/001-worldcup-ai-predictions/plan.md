## Summary

Build a full-stack AI system that predicts 2026 World Cup match outcomes (Win/Draw/Loss
probabilities + expected score) and displays them on a live-updating dashboard. Pre-match
predictions use a calibrated XGBoost classifier trained on historical international match
data. Once a match starts, a Poisson in-game model updates probabilities within 30 seconds
of each live event. The React frontend connects via WebSocket to a FastAPI backend hosted
on Railway; the frontend is deployed on Vercel. PostgreSQL on Railway stores all match
data, predictions, and accuracy records.

## Architecture Deep-Dive

### Data Flow (Live Match)

```
1. APScheduler (asyncio task, 15s interval)
       в”‚ polls FootballDataProvider.get_match_state(fixture_id)
       в†“
2. Ingestion Service
       в”‚ normalizes response via adapter
       в”‚ deduplicates via external_event_id (ON CONFLICT DO NOTHING)
       в”‚ inserts new live_events rows
       в†“ (only if new event inserted)
3. Prediction Engine
       в”‚ loads latest live_events for this match
       в”‚ computes in-game features (score diff, cards, minute)
       в”‚ runs Poisson update: О»_remaining Г— Poisson PMF enumeration
       в”‚ blends with pre-match XGBoost output
       в”‚ generates SHAP factors for top-3
       в”‚ inserts predictions row
       в†“
4. Broadcast Service
       в”‚ ConnectionManager.broadcast_to_match(match_id, prediction)
       в†“
5. All WebSocket clients subscribed to this match_id
       receive { "type": "prediction_update", "payload": {...} }
       в†“
6. React (Zustand store update в†’ component re-render в†’ animated probability bars)
```

### ML Prediction Pipeline

```
OFFLINE (training, run once):
  Historical data (openfootball + Kaggle)
       в†“ features.py (feature engineering)
  Feature matrix X + labels y
       в†“ train_prematch.py
  XGBoostClassifier(max_depth=4, n_estimators=300, early_stopping=50)
       в†“ CalibratedClassifierCV(method='isotonic', cv=5)
  Calibrated model в†’ bootstrap 200x в†’ CI offsets per stage
       в†“ artifacts/prematch_v1.0.0.joblib

ONLINE (inference, per match):
  Pre-match features (FIFA rank, form, H2H)
       в†“ calibrated_model.predict_proba(X)
  [P(home_win), P(draw), P(away_win)] + xG predictions
       в†“ shap.TreeExplainer в†’ top-3 factors
  Pre-match prediction stored + WebSocket broadcast

  [Live event arrives]
       в†“ update in-game features (score, cards, minute)
  Poisson model:
    О»_remaining = prematch_xG Г— (1 в€’ minute/90) Г— trailing_factor
    P(final score h:a) = ОЈ Poisson(hв€’h0, О»_home) Г— Poisson(aв€’a0, О»_away)
    P(home_win) = ОЈ_{h>a} P(final h:a)
       в†“ blend: w_poisson = minute/90, w_prematch = 1 в€’ minute/90
  Updated prediction stored + WebSocket broadcast
```

### WebSocket Connection Manager

```python
# Key implementation (from research):
class ConnectionManager:
    def __init__(self):
        self.rooms: dict[str, set[WebSocket]] = defaultdict(set)

    async def connect(self, match_id: str, ws: WebSocket):
        await ws.accept()
        self.rooms[match_id].add(ws)
        await ws.send_json({"type": "connected", "match_id": match_id, ...})

    def disconnect(self, match_id: str, ws: WebSocket):
        self.rooms[match_id].discard(ws)
        if not self.rooms[match_id]:
            del self.rooms[match_id]

    async def broadcast_to_match(self, match_id: str, data: dict):
        dead = set()
        for ws in self.rooms.get(match_id, set()):
            try:
                await ws.send_json(data)
            except Exception:
                dead.add(ws)
        for ws in dead:
            self.disconnect(match_id, ws)
```

Scheduler uses `asyncio.create_task()` inside `lifespan()` вЂ” not APScheduler вЂ” to
avoid silent exception swallowing.

## Complexity Tracking

No constitution violations. Architecture is appropriately scoped:

| Decision | Rationale |
|----------|-----------|
| Two ML models (not one) | Required by data integrity вЂ” single model with zero-padded live features creates systematic kickoff bias |
| WebSocket per match (not global) | Correct subscription granularity; avoids broadcasting irrelevant match updates to all clients |
| Adapter pattern for data provider | Constitution IV (modular) + enables free mock provider for testing without API key |

## Key Risks and Mitigations

| Risk | Likelihood | Impact | Mitigation |
|------|-----------|--------|-----------|
| ESPN unofficial API changes endpoint or breaks | Medium | High | `FootballDataProvider` ABC isolates ESPN logic to `espn.py` вЂ” swap to `api_football.py` is one config change; monitor with `/health` endpoint |
| ESPN deduplication fails (no stable event IDs) | Medium | Medium | Construct `external_event_id` from `{gameId}_{minute}_{type}_{team}_{player}` composite key; UNIQUE constraint prevents double-trigger |
| Railway WebSocket connections dropped by idle timeout | High | Medium | Server-side 30s JSON ping (researched and confirmed pattern) |
| ML model underperforms for tournament first-timers (debut matchups) | Low | Medium | `h2h_matches_count` feature + "Limited data" badge when < 3 H2H meetings |
| XGBoost calibration fails with < 500 draw samples | Medium | Low | Fall back to Platt scaling if Draw class has < 500 samples after bootstrap |
| Vercel build fails if VITE_WS_URL not set | High | High | `.env.example` documents all required vars; checked before deploy |
| StatsBomb Open Data doesn't cover all needed competitions | Low | Low | Fall back to raw goals_avg if xG unavailable; feature is additive not critical |
